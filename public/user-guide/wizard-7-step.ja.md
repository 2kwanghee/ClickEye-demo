# 7-Step ソリューションウィザードの使い方

ClickEye のウィザードは全 10 画面で構成されており、このうち 2 画面は AI が自動で処理します。ユーザーが直接入力・選択する画面は 8 画面です。

---

## 全体の流れ

```
Step 1. 会社情報の入力
   ↓ (次へをクリック)
[AI 自動] ソリューションプロトタイプ生成 (~30秒)
   ↓
Step 2. プロトタイプ選択
   ↓ (次へをクリック)
[AI 自動] PM 推薦分析 (~10秒)
   ↓
Step 3. PM 選択
   ↓
Step 4. PM 構成の確認
   ↓
Step 5. エージェント & スキル設定
   ↓
Step 6. プラットフォーム選択
   ↓
Step 7. 環境変数の入力
   ↓
Step 8. 最終確認 & プロジェクト生成
```

---

## Step 1 — 会社情報 & ソリューション要件

AI がソリューションを設計するために、会社の背景と要件を把握するステップです。

### 入力項目

| 項目 | 必須 | 制限 | 説明 |
|------|------|------|------|
| 会社名 | ✅ | 200文字 | 会社またはチーム名 |
| 会社規模 | ✅ | 選択 | スタートアップ（1〜10名）/ 小規模企業（11〜50名）/ 中小企業（51〜200名）/ 中堅企業（201〜1,000名）/ 大企業（1,000名以上） |
| 業種 | ✅ | 選択 | IT、金融/フィンテック、Eコマース、ヘルスケア、教育、製造業、物流、マーケティング、ゲーム、その他 |
| 技術スタック | ⬜ | 複数選択 | Python、TypeScript、React、Next.js、FastAPI、Docker など（複数選択可能） |
| 主力の製品/サービス | ✅ | 500文字 | 現在、会社が販売・運営中の製品またはサービス |
| ビジネスタイプ | ✅ | 選択 | B2B / B2C / B2B2C / 社内ツール |
| 会社説明 | ⬜ | 1,000文字 | チーム構成、開発レベル、現在の課題などの追加コンテキスト |
| 必要なソリューションの説明 | ✅ | 最小 50文字 / 最大 2,000文字 | **AI が最も重視して参照する項目** |

### 記入例

**「必要なソリューションの説明」の上手な書き方:**

```
✅ 良い例:
現在 Excel で管理している顧客問い合わせを自動化したいです。
営業チーム 8 名が 1 日 100 件以上の問い合わせを処理しており、繰り返しの質問には自動で回答し、
緊急の案件は Slack で通知が来ると助かります。
CRM と連携できればさらに良いです。

❌ 悪い例:
AI 自動化が必要です。
```

### よくある失敗

- **ソリューションの説明が短すぎる**: 最小 50 文字未満では「次へ」ボタンが有効になりません。具体的な業務フローと望む結果を記入してください。
- **技術スタックの未選択**: 選択しなくても進められますが、現在チームが使用している技術を選択すると、AI がより適したソリューションを設計します。
- **会社説明の省略**: 必須項目ではありませんが、チーム規模・現在の技術レベル・制約事項を記入すると、はるかに正確なプロトタイプが生成されます。

---

## [AI 自動] ソリューションプロトタイプ生成

「次へ」ボタンを押すと、AI が自動で 3 つのソリューションプロトタイプを生成します。約 30 秒かかります。この画面ではユーザーの入力は不要です。

---

## Step 2 — プロトタイプ選択

AI が生成した 3 つのソリューション候補の中から、プロジェクトの方向性に最も合うものを選択します。

### カードの読み方

各プロトタイプのカードには、次の情報が表示されます:

- **名称**: ソリューション名
- **タイプ**: SaaS / REST API / フルスタック / 社内ツール / MVP / カスタム
- **説明**: ソリューションの概要

### よくある失敗

- **「最も高機能に見えるもの」を選ぶ**: 複雑なソリューションが常に良いとは限りません。チーム規模と技術レベルに合ったものを選んでください。
- **どれも気に入らない場合**: 前のステップに戻り、ソリューション要件をより具体的に記入してから再生成してください。

---

## [AI 自動] PM 推薦分析

プロトタイプの選択後、AI がそのソリューションに最適化された PM（Project Manager AI）の一覧を分析します。約 10 秒かかります。

---

## Step 3 — PM 選択

AI が推薦した PM の一覧から、プロジェクトを率いる PM AI を選択します。

### PM カードの読み方

| 項目 | 説明 |
|------|------|
| アバター & 名前 | PM プロフィール |
| ドメイン | PM が専門とする分野 |
| 一致率（%） | 選択したプロトタイプとの適合度 |
| 推薦理由 | AI 分析の根拠 |
| 成功率 / 完了プロジェクト / 平均評価 | 実績指標 |

### よくある失敗

- **一致率だけで選ぶ**: ドメインがプロジェクトの業種に合っているかも確認してください。
- **選択せずに次へ**: PM の選択は必須です。未選択の場合は「次へ」ボタンが無効になります。

---

## Step 4 — PM 構成の確認

選択した PM が、どのサブエージェントとスキルで構成されているかを確認するステップです。修正せず、確認だけして次に進んでも構いません。

---

## Step 5 — エージェント & スキル設定

PM 構成をもとに、デフォルトのエージェントが選択されています。必要に応じて追加・削除できます。

### エージェント

AI エージェントはそれぞれ特定の役割（コード作成、テスト、ドキュメント化など）を担当します。デフォルトで選択された項目をそのまま維持することを推奨します。

### 連携スキル（任意）

スキルは外部サービスとの連携機能です。

| スキル | 説明 | 追加設定の要否 |
|------|------|--------------|
| linear | Linear のイシュートラッキング連携 | ✅ LINEAR_API_KEY、LINEAR_TEAM_ID |
| slack | Slack への通知送信 | ✅ SLACK_WEBHOOK_URL |
| github | GitHub の PR 自動生成 | ✅ GITHUB_TOKEN |

> **Linear スキルを選択した場合**: Step 7（環境変数）で `LINEAR_API_KEY` と `LINEAR_TEAM_ID` が必須として追加されます。[Linear 連携設定ガイド](/guide/linear-integration-setup)を参照してください。

### よくある失敗

- **不要なスキルの追加**: スキルを追加すると、その API キーも必須で入力する必要があります。使用しないサービスのスキルは追加しないでください。

---

## Step 6 — プラットフォーム選択

ダウンロードした ZIP を実行する AI エージェントプラットフォームを選択します。

| プラットフォーム | 推奨対象 | 備考 |
|--------|----------|------|
| **Claude Code** ⭐ | ほとんどのユーザー | Anthropic の AI コーディングエージェント、最もよくサポートされています |
| Gemini CLI | Google エコシステムのユーザー | Google Gemini ベース |
| Cursor | AI コードエディタを好む方 | VS Code ベースのエディタ |
| Codex | OpenAI ユーザー | OpenAI Codex ベース |

### よくある失敗

- **ANTHROPIC_API_KEY がないのに Claude Code を選択**: Claude Code を選択すると、Step 7 で必ず Anthropic API キーを入力する必要があります。[console.anthropic.com](https://console.anthropic.com) で事前に発行してください。

---

## Step 7 — 環境変数の設定

ソリューションの実行に必要な API キーと環境変数を入力します。ここで入力した値は、ZIP ファイル内の `.env` ファイルに保存されます。

### 必須の API キー

| キー | 必須かどうか | 発行元 |
|----|----------|-------|
| `ANTHROPIC_API_KEY` | 常に必須 | [console.anthropic.com](https://console.anthropic.com) |
| `LINEAR_API_KEY` | Linear スキル選択時 | [linear.app/settings/api](https://linear.app/settings/api) |
| `LINEAR_TEAM_ID` | Linear スキル選択時 | Linear のチーム URL で確認 |

### Linear スキル選択時 — リアルタイムトラッキング方式

| 方式 | コスト | 特徴 | 推奨対象 |
|------|------|------|----------|
| **Cloudflare Tunnel** ⭐ | 無料 | 静的 URL、ターミナルの維持が必要 | ほとんどのユーザー |
| ngrok | 有料 $8/月（固定）/ 無料（一時） | 再起動時に URL が変更（無料） | ngrok アカウント保有者 |
| 30秒ポーリング | 無料 | webhook 不要、最大 30 秒の遅延 | トンネル設定が難しい場合 |

### 環境変数の追加方法

1. 画面下部の **環境変数の追加** セクションで、キー名（`KEY_NAME`）と値を入力します。
2. **+** ボタンをクリックするか、Enter キーを押します。
3. 値を後で入力したい場合は、ZIP の解凍後に `.env` ファイルを直接編集してください。

### よくある失敗

- **`ANTHROPIC_API_KEY` を未入力のまま次へ**: 必須キーが未設定の状態でも次のステップへ進めますが、ZIP の実行時にエラーが発生します。**必ず入力してから進めて**ください。
- **ZIP ファイルの共有**: `.env` ファイルには実際のキーが含まれます。ZIP ファイルを他人に共有しないでください。
- **ngrok 無料プランで Cloudflare 方式を選択**: 再起動時に URL が変わり、webhook が無効化されます。ClickEye の設定ページで URL を再度保存する必要があります。

---

## Step 8 — 最終確認 & プロジェクト生成

これまでに入力したすべての設定を要約して表示します。問題がなければ **プロジェクト生成** ボタンをクリックしてください。

### 確認項目

- 会社名 / ビジネスタイプ / 主力製品
- 選択したプロトタイプ名 & タイプ
- 選択した PM
- エージェント & スキルの一覧
- プラットフォーム
- 設定された環境変数の数

### プロジェクト生成後

1. ZIP ファイルのダウンロードボタンが表示されます。
2. ZIP を解凍し、選択したプラットフォーム（Claude Code など）を起動します。
3. AI 開発が始まります。

---

## よくある質問

**Q. 前のステップに戻ると AI は再生成しますか？**
A. 会社情報（Step 1）を変更した場合にのみ、プロトタイプが再生成されます。それ以外のステップは、前のステップに戻っても再生成されません。

**Q. ウィザードの途中で閉じるとどうなりますか？**
A. ブラウザのセッションが維持されている間は、入力内容が保存されます。ブラウザを完全に閉じると、最初からやり直す必要があります。

**Q. 生成された ZIP にはどのようなファイルが入っていますか？**
A. 選択したプラットフォームに合わせた設定ファイル（`.claude/`、`.gemini/` など）、`.env`、`CLAUDE.md`、`README.md` などが含まれます。
