# 7-Step 솔루션 위저드 사용법

ClickEye 위저드는 총 10개 화면으로 구성되어 있으며, 이 중 2개는 AI가 자동으로 처리합니다. 사용자가 직접 입력/선택하는 화면은 8개입니다.

---

## 전체 흐름

```
Step 1. 회사 정보 입력
   ↓ (다음 클릭)
[AI 자동] 솔루션 프로토타입 생성 (~30초)
   ↓
Step 2. 프로토타입 선택
   ↓ (다음 클릭)
[AI 자동] PM 추천 분석 (~10초)
   ↓
Step 3. PM 선택
   ↓
Step 4. PM 구성 확인
   ↓
Step 5. 에이전트 & 스킬 설정
   ↓
Step 6. 플랫폼 선택
   ↓
Step 7. 환경변수 입력
   ↓
Step 8. 최종 확인 & 프로젝트 생성
```

---

## Step 1 — 회사 정보 & 솔루션 요구사항

AI가 솔루션을 설계하기 위해 회사 맥락과 요구사항을 파악하는 단계입니다.

### 입력 항목

| 항목 | 필수 | 제한 | 설명 |
|------|------|------|------|
| 회사 이름 | ✅ | 200자 | 회사 또는 팀 이름 |
| 회사 규모 | ✅ | 선택 | 스타트업(1–10명) / 소기업(11–50명) / 중소기업(51–200명) / 중견기업(201–1,000명) / 대기업(1,000명+) |
| 업종 | ✅ | 선택 | IT, 금융/핀테크, 이커머스, 헬스케어, 교육, 제조업, 물류, 마케팅, 게임, 기타 |
| 기술 스택 | ⬜ | 복수 선택 | Python, TypeScript, React, Next.js, FastAPI, Docker 등 (중복 선택 가능) |
| 주력 제품/서비스 | ✅ | 500자 | 현재 회사가 판매/운영 중인 제품 또는 서비스 |
| 비즈니스 유형 | ✅ | 선택 | B2B / B2C / B2B2C / 내부 도구 |
| 회사 설명 | ⬜ | 1,000자 | 팀 구성, 개발 수준, 현재 문제 등 추가 맥락 |
| 필요한 솔루션 설명 | ✅ | 최소 50자 / 최대 2,000자 | **AI가 가장 중요하게 참고하는 항목** |

### 작성 예시

**필요한 솔루션 설명 잘 쓰는 법:**

```
✅ 좋은 예:
현재 엑셀로 관리하는 고객 문의를 자동화하고 싶습니다.
영업팀 8명이 하루 100건 이상의 문의를 처리하는데, 반복 질문에 자동 답변하고
긴급 건은 Slack으로 알림이 오면 좋겠습니다.
CRM과 연동되면 더욱 좋습니다.

❌ 나쁜 예:
AI 자동화가 필요합니다.
```

### 흔한 실수

- **솔루션 설명을 너무 짧게 작성**: 최소 50자 미만이면 다음 버튼이 활성화되지 않습니다. 구체적인 업무 흐름과 원하는 결과를 적어주세요.
- **기술 스택 미선택**: 선택하지 않아도 진행 가능하지만, 현재 팀이 사용 중인 기술을 선택하면 AI가 더 적합한 솔루션을 설계합니다.
- **회사 설명 생략**: 필수 항목은 아니지만, 팀 규모·현재 기술 수준·제약 사항을 적으면 훨씬 정확한 프로토타입이 생성됩니다.

---

## [AI 자동] 솔루션 프로토타입 생성

다음 버튼을 누르면 AI가 자동으로 3개의 솔루션 프로토타입을 생성합니다. 약 30초 소요됩니다. 이 화면에서는 사용자 입력이 필요 없습니다.

---

## Step 2 — 프로토타입 선택

AI가 생성한 3개의 솔루션 후보 중 프로젝트 방향과 가장 잘 맞는 것을 선택합니다.

### 카드 읽는 법

각 프로토타입 카드에는 다음 정보가 표시됩니다:

- **이름**: 솔루션 명칭
- **유형**: SaaS / REST API / 풀스택 / 내부 도구 / MVP / 커스텀
- **설명**: 솔루션 개요

### 흔한 실수

- **"가장 비싸 보이는 것" 선택**: 복잡한 솔루션이 항상 좋은 것은 아닙니다. 팀 규모와 기술 수준에 맞는 것을 선택하세요.
- **모두 마음에 안 든다면**: 이전 단계로 돌아가 솔루션 요구사항을 더 구체적으로 작성한 뒤 재생성하세요.

---

## [AI 자동] PM 추천 분석

프로토타입 선택 후, AI가 해당 솔루션에 최적화된 PM(Project Manager AI) 목록을 분석합니다. 약 10초 소요됩니다.

---

## Step 3 — PM 선택

AI가 추천한 PM 목록에서 프로젝트를 이끌 PM AI를 선택합니다.

### PM 카드 읽는 법

| 항목 | 설명 |
|------|------|
| 아바타 & 이름 | PM 프로필 |
| 도메인 | PM이 전문으로 하는 분야 |
| 일치율(%) | 선택한 프로토타입과의 적합도 |
| 추천 이유 | AI 분석 근거 |
| 성공률 / 완료 프로젝트 / 평균 평점 | 실적 지표 |

### 흔한 실수

- **일치율만 보고 선택**: 도메인이 프로젝트 업종과 맞는지도 확인하세요.
- **선택 안 하고 다음**: PM 선택은 필수입니다. 미선택 시 다음 버튼이 비활성화됩니다.

---

## Step 4 — PM 구성 확인

선택한 PM이 어떤 서브 에이전트와 스킬로 구성되어 있는지 확인하는 단계입니다. 수정 없이 확인만 하고 다음으로 넘어가도 됩니다.

---

## Step 5 — 에이전트 & 스킬 설정

PM 구성을 기반으로 기본 에이전트가 선택되어 있습니다. 필요에 따라 추가하거나 제거할 수 있습니다.

### 에이전트

AI 에이전트는 각자 특정 역할(코드 작성, 테스트, 문서화 등)을 담당합니다. 기본 선택된 항목을 유지하는 것을 권장합니다.

### 연동 스킬 (선택)

스킬은 외부 서비스와의 연동 기능입니다.

| 스킬 | 설명 | 추가 설정 필요 |
|------|------|--------------|
| linear | Linear 이슈 추적 연동 | ✅ LINEAR_API_KEY, LINEAR_TEAM_ID |
| slack | Slack 알림 전송 | ✅ SLACK_WEBHOOK_URL |
| github | GitHub PR 자동 생성 | ✅ GITHUB_TOKEN |

> **Linear 스킬 선택 시**: Step 7(환경변수)에서 `LINEAR_API_KEY`와 `LINEAR_TEAM_ID`가 필수로 추가됩니다. [Linear 연동 설정 가이드](/guide/linear-integration-setup)를 참고하세요.

### 흔한 실수

- **필요 없는 스킬 추가**: 스킬을 추가하면 해당 API 키도 필수로 입력해야 합니다. 사용하지 않을 서비스의 스킬은 추가하지 마세요.

---

## Step 6 — 플랫폼 선택

다운로드한 ZIP을 실행할 AI 에이전트 플랫폼을 선택합니다.

| 플랫폼 | 권장 대상 | 비고 |
|--------|----------|------|
| **Claude Code** ⭐ | 대부분의 사용자 | Anthropic의 AI 코딩 에이전트, 가장 잘 지원됨 |
| Gemini CLI | Google 생태계 사용자 | Google Gemini 기반 |
| Cursor | AI 코드 에디터 선호자 | VS Code 기반 에디터 |
| Codex | OpenAI 사용자 | OpenAI Codex 기반 |

### 흔한 실수

- **ANTHROPIC_API_KEY가 없는데 Claude Code 선택**: Claude Code를 선택하면 Step 7에서 반드시 Anthropic API 키를 입력해야 합니다. [console.anthropic.com](https://console.anthropic.com)에서 미리 발급받으세요.

---

## Step 7 — 환경변수 설정

솔루션 실행에 필요한 API 키와 환경변수를 입력합니다. 여기서 입력한 값은 ZIP 파일 내 `.env` 파일에 저장됩니다.

### 필수 API 키

| 키 | 필수 여부 | 발급처 |
|----|----------|-------|
| `ANTHROPIC_API_KEY` | 항상 필수 | [console.anthropic.com](https://console.anthropic.com) |
| `LINEAR_API_KEY` | Linear 스킬 선택 시 | [linear.app/settings/api](https://linear.app/settings/api) |
| `LINEAR_TEAM_ID` | Linear 스킬 선택 시 | Linear 팀 URL에서 확인 |

### Linear 스킬 선택 시 — 실시간 트래킹 방식

| 방식 | 비용 | 특징 | 권장 대상 |
|------|------|------|----------|
| **Cloudflare Tunnel** ⭐ | 무료 | 정적 URL, 터미널 유지 필요 | 대부분의 사용자 |
| ngrok | 유료 $8/월 (고정) / 무료 (임시) | 재시작 시 URL 변경 (무료) | ngrok 계정 보유자 |
| 30초 폴링 | 무료 | webhook 불필요, 최대 30초 지연 | 터널 설정 어려운 경우 |

### 환경변수 추가 방법

1. 화면 하단 **환경변수 추가** 섹션에서 키 이름(`KEY_NAME`)과 값을 입력합니다.
2. **+** 버튼을 클릭하거나 Enter 키를 누릅니다.
3. 값을 나중에 입력하고 싶다면 ZIP 압축 해제 후 `.env` 파일을 직접 수정하세요.

### 흔한 실수

- **`ANTHROPIC_API_KEY` 미입력 후 다음**: 필수 키가 미설정된 상태에서도 다음 단계로 이동할 수 있지만, ZIP 실행 시 오류가 발생합니다. **반드시 입력 후 진행**하세요.
- **ZIP 파일 공유**: `.env` 파일에 실제 키가 포함됩니다. ZIP 파일을 타인에게 공유하지 마세요.
- **ngrok 무료 플랜으로 Cloudflare 방식 선택**: 재시작 시 URL이 바뀌어 webhook이 무효화됩니다. ClickEye 설정 페이지에서 URL을 다시 저장해야 합니다.

---

## Step 8 — 최종 확인 & 프로젝트 생성

지금까지 입력한 모든 설정을 요약해서 보여줍니다. 이상이 없으면 **프로젝트 생성** 버튼을 클릭하세요.

### 확인 항목

- 회사명 / 비즈니스 유형 / 주력 제품
- 선택한 프로토타입 이름 & 유형
- 선택한 PM
- 에이전트 & 스킬 목록
- 플랫폼
- 설정된 환경변수 개수

### 프로젝트 생성 후

1. ZIP 파일 다운로드 버튼이 표시됩니다.
2. ZIP을 압축 해제하고 선택한 플랫폼(Claude Code 등)을 실행합니다.
3. AI 개발이 시작됩니다.

---

## 자주 묻는 질문

**Q. 이전 단계로 돌아가면 AI가 재생성하나요?**
A. 회사 정보(Step 1) 변경 시에만 프로토타입이 재생성됩니다. 나머지 단계는 이전 단계로 돌아가도 재생성하지 않습니다.

**Q. 위저드 중간에 닫으면 어떻게 되나요?**
A. 브라우저 세션이 유지되는 동안 입력 내용이 보존됩니다. 브라우저를 완전히 닫으면 처음부터 다시 시작해야 합니다.

**Q. 생성된 ZIP에 어떤 파일이 들어있나요?**
A. 선택한 플랫폼에 맞는 설정 파일(`.claude/`, `.gemini/` 등), `.env`, `CLAUDE.md`, `README.md` 등이 포함됩니다.
